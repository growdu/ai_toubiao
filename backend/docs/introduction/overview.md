# 产品概览

## 一句话定义

> **BidWriter** —— 让中小型投标团队 4 小时内交付一份 80% 完成度的高合规标书。

## 它解决什么问题

### 传统标书写作的痛点

- 😩 **重复劳动** —— 同一份资质材料、案例描述、技术方案框架反复改
- 😩 **新人难上手** —— 老员工经验沉淀在个人电脑里，无法复用
- 😩 **AI 切换成本高** —— DeepSeek 便宜但弱，Claude 强但贵，GPT 综合好但中规中矩
- 😩 **协作混乱** —— 多人靠微信传文件，版本靠"投标专用 - 终版 - 真的不改了.docx"
- 😩 **废标风险高** —— 响应条款、资质要求、错别字、逻辑矛盾，分散在 100+ 页文档里

### 我们怎么解

| 痛点 | BidWriter 解法 |
|---|---|
| 重复劳动 | 知识库 + 全局事实 + 模板复用 |
| 新人上手难 | 提示词沉淀 + 行业模板 + AI 协作 |
| AI 切换成本 | 多模型路由（成本/质量自适应） |
| 协作混乱 | 团队空间 + 评审流 + 操作审计 |
| 废标风险 | 四层一致性审计（废标/错别字/逻辑/查重） |

## 核心能力

### 🧩 五步法工作流

```mermaid
graph LR
    A[招标上传] --> B[Step02 解析]
    B --> C[Step03 拆解大纲]
    C --> D[Step04 全局事实]
    D --> E[Step05 正文生成]
    E --> F[废标检查]
    F --> G[Word 导出]
```

详细：[架构 / 状态机](../architecture/state-machine.md)

### 🤖 多模型路由（核心创新）

每次 AI 调用根据**任务画像 + 质量要求 + 成本预算 + 降级链**自动选模型：

| 任务 | 主模型 | 备选 |
|---|---|---|
| 招标解析 | Claude Sonnet 4 | GPT-4o → DeepSeek |
| 大纲生成 | DeepSeek Chat | GPT-4o-mini |
| 正文生成 | Claude Sonnet 4 | GPT-4o |
| 一致性审计 | Claude Sonnet 4 | GPT-4o |
| 配图 | DALL·E 3 | Stable Diffusion |

详细：[架构 / AI 路由](../architecture/ai-router.md)

### 🔍 三层知识库

```
Layer 1  精确匹配（目录树 + 关键词全文）
Layer 2  弱 RAG（pgvector 向量 + BM25 重排序）
Layer 3  全局事实（从历史标书抽取的强约束）
```

详细：[架构 / 模块设计 - 知识库](../architecture/modules.md)

### 🛡️ 四层一致性审计

| 层 | 检查内容 |
|---|---|
| L1 废标项 | 必答条款、强制资质、否决项 |
| L2 一致性 | 跨章节数据冲突、参数矛盾 |
| L3 错别字 | 字词错误、标点、敏感词 |
| L4 查重 | 与历史标书雷同度 |

详细：[架构 / 模块设计 - 审计](../architecture/modules.md#audit-svc)

## 适用场景

✅ **最适合**：
- 信息化系统集成标（最大市场）
- 政府采购
- 工程 EPC
- 咨询服务
- 设备采购
- 教育培训

❌ **暂不适合**：
- 国家级特殊行业标（涉密、军品）
- 需要专家现场答辩的标（需人工服务配合）
- 标书金额 < 10 万的小标（成本不划算）

## 与 yibiao 的差异

本项目在架构设计上借鉴了 [OpenBidKit_Yibiao](https://github.com/FB208/OpenBidKit_Yibiao)（易标投标工具箱）的工程经验，但**不是照搬**。

| 维度 | yibiao | BidWriter |
|---|---|---|
| 形态 | 桌面单机 | Web SaaS + 桌面备 |
| 用户 | 单用户 | 多租户 / 团队 |
| AI | 单模型自配 | 多模型路由 |
| 计费 | 免费开源 | 按订阅 + Token |
| 协作 | 无 | 实时协作 |
| 部署 | 用户本地 | 云端 + 私有化双模 |

借鉴的工程经验：
- Step01-05 工作流抽象
- 任务状态机设计
- 全文一致性审计
- Prompt 缓存与 JSON 修复

详细对比：[introduction/comparison.md](comparison.md)
