# 迭代路线

> **M1（最小可用）→ M4（生态完善）** —— 四个里程碑，每个 8 周左右。

---

## 总览

| 里程碑 | 时间 | 目标 | 关键交付 |
|---|---|---|---|
| **M1** | 第 1-8 周 | 单租户 MVP | 工作流 + 单 AI + Word 导出 |
| **M2** | 第 9-16 周 | 多租户 + 团队协作 | RBAC + 知识库 + 路由表 |
| **M3** | 第 17-24 周 | 多模型 + UGC 模板 | 隐式反馈 + 模板市集 + 桌面 |
| **M4** | 第 25-32 周 | 生态 + 企业版 | 付费模板 + 私有化 + SSO |
| **v1.0 GA** | 第 33 周 | 正式发布 | SLA + 全文档 + 商业化 |

---

## M1：MVP（最小可用）

**目标**：证明"AI 写标书"核心价值闭环

### 功能

- ✅ 用户注册 / 登录（邮箱密码）
- ✅ 创建项目（单租户）
- ✅ 上传招标文档（PDF / docx）
- ✅ 文档解析 → Markdown
- ✅ 工作流 Step01-05
- ✅ 单 AI Provider（OpenAI GPT-4o）
- ✅ 大纲编辑
- ✅ 正文生成
- ✅ 一致性审计（normal 模式）
- ✅ Word 导出（.docx）
- ✅ 基础 UI（Next.js）

### 技术

- ✅ Go API（单体，不分服务）
- ✅ PostgreSQL + pgvector
- ✅ Redis（任务队列）
- ✅ OpenAI API 集成

### 验收标准

- [ ] 一个用户能独立完成"创建项目 → 上传招标 → 生成标书 → 导出 Word"全流程
- [ ] 单标耗时 < 4 小时（vs 行业 2-3 天）
- [ ] 标书基础质量评分 > 80（人工评估）
- [ ] 系统 uptime > 95%

### 团队

- 1 后端 + 1 前端 + 1 全栈（你）+ 1 产品 + 1 设计

### 风险

- ⚠️ AI 输出质量不稳定 → 借鉴 yibiao 的 prompt 工程
- ⚠️ 大文档处理慢 → 流式 + 分块
- ⚠️ 文档解析不准确 → 多重 fallback（pdf → docx → 文本提取）

---

## M2：多租户 + 协作

**目标**：服务中小团队（5-50 人）

### 功能

- ✅ 多租户隔离（行级 tenant_id）
- ✅ 团队空间（成员管理）
- ✅ RBAC 4 角色
- ✅ 实时协作（SSE / WebSocket）
- ✅ 评论与@提醒
- ✅ 知识库（3 层：精确 + RAG + 全局事实）
- ✅ 模板系统（6 个行业预置）
- ✅ 规则路由（多 Provider 备选）
- ✅ Token 用量统计

### 技术

- ✅ 微服务拆分（10 个服务）
- ✅ API Gateway
- ✅ Asynq 任务队列
- ✅ Prometheus + Grafana
- ✅ Sentry 错误监控

### 验收标准

- [ ] 100 租户同时在线无性能问题
- [ ] 5 人同时协作无冲突
- [ ] 多模型自动降级可用（主失败 → 备成功）
- [ ] 知识库搜索准确率 > 70%
- [ ] 系统 uptime > 99%

### 团队

- 2 后端 + 2 前端 + 1 全栈 + 1 产品 + 1 设计 + 1 测试

---

## M3：智能增强 + 桌面

**目标**：建立差异化（智能 + 多端）

### 功能

- ✅ 隐式反馈（采纳率 / 修改率 / 重生成）
- ✅ 路由表动态调整
- ✅ agent 模式深度审计（OpenCode 子进程）
- ✅ 模板市集（UGC）
- ✅ 桌面客户端（Tauri，Mac 先行）
- ✅ 数据同步（云端权威 + 桌面缓存）
- ✅ 自动启用 agent 规则
- ✅ Mermaid 流程图渲染
- ✅ 配图生成（DALL·E 3 / Stable Diffusion）

### 技术

- ✅ LLM-as-a-Judge 抽样评估
- ✅ Tauri 2 桌面框架
- ✅ SQLite + 同步队列
- ✅ 离线优先架构

### 验收标准

- [ ] 桌面端 macOS GA
- [ ] 路由表根据真实反馈动态调整
- [ ] agent 模式对关键标段 issue 检出率 > 95%
- [ ] UGC 模板 100+（含审核）
- [ ] 用户活跃度（DAU/MAU）> 30%

### 团队

- 3 后端 + 2 前端 + 1 桌面 + 1 全栈 + 1 AI + 1 产品 + 1 设计 + 2 测试

---

## M4：企业版 + 生态

**目标**：服务企业客户 + 建立生态

### 功能

- ✅ 私有化部署（Helm）
- ✅ 企业 SSO（SAML 2.0）
- ✅ MFA（TOTP / WebAuthn）
- ✅ 审计日志（不可变）
- ✅ 付费模板市集
- ✅ Webhook 集成
- ✅ SDK（TypeScript / Go / Python）
- ✅ 开放 API（OpenAPI）
- ✅ Windows / Linux 桌面

### 技术

- ✅ Helm Chart（生产级）
- ✅ mTLS 内部通信
- ✅ Vault 密钥管理
- ✅ SOC 2 合规启动

### 验收标准

- [ ] 私有化客户 5+ 上线
- [ ] 企业 SSO 集成 < 1 天
- [ ] 付费模板 50+
- [ ] API 调用成功率 > 99.9%
- [ ] 客户 NPS > 40

### 团队

- 4 后端 + 3 前端 + 2 桌面 + 1 AI + 1 DevOps + 2 产品 + 1 设计 + 1 安全 + 3 测试

**目标**：商业化 + SLA + 完整产品

### 功能

- ✅ 完整文档 + 培训
- ✅ 客户支持体系
- ✅ 商业化（订阅 + Token）
- ✅ 数据分析（产品迭代）
- ✅ 灰度发布
- ✅ A/B 测试框架

### 技术

- ✅ 多区域部署（华东 + 华北 + 华南）
- ✅ 灾备（同城双活）
- ✅ 性能优化（p99 < 500ms）
- ✅ 安全加固（渗透测试通过）

### 验收标准

- [ ] SLA 99.9% 持续 90 天
- [ ] 付费客户 100+
- [ ] MRR ¥50 万

---

## 不做什么（M1-M4 范围内）

❌ **不做**：

- 国家级特殊行业（军品、涉密）—— 走定制
- 实时多人协同编辑同一段落（v2.0+）—— 用 CRDT
- 自研 AI 模型 —— 用开源 / API
- 多语言（v1.0 仅中文）
- 移动 App —— 用响应式 Web
