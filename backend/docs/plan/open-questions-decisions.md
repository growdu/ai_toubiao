## Q1：多租户隔离粒度

**推荐：行级隔离（tenant_id 字段），M1-M3 长期使用**

理由：
- 中小企业规模（5-50 人/租户），行级完全够用
- 单库管理运维成本低：备份、迁移、监控一套搞定
- 跨租户分析（行业洞察、产品迭代）容易做
- pgvector 索引、Redis 缓存可共享

备选：schema-per-tenant
- 切换条件：客户量 > 100 租户 且 单租户数据 > 10 GB
- 切换成本：迁移脚本 + ORM 改造 + 租户感知路由

实施细节：
- 所有业务表强制带 tenant_id（外键 + NOT NULL）
- 中间件层强制注入：每个 SQL 必须带 tenant_id
- GORM 用 BeforeQuery hook + sqlc 模板双重保险
- 自动化测试：每个 API 必须有"跨租户访问应拒绝"的 case

退出条件：
- 单 PG 实例 > 500 GB
- 某租户合规要求物理隔离（金融/政府）
- 跨租户查询成为性能瓶颈

## Q2：模型路由的"质量历史"如何积累

**推荐：M1-M2 用规则路由（任务画像 → 路由表），M3 引入隐式反馈（用户行为），M4+ 引入 LLM-as-a-Judge**

理由：
- 路由冷启动需要数据；规则路由先用起来，让真实流量积累数据
- 用户显式反馈（点赞/点踩）成本高，样本稀疏
- 用户隐式反馈（采纳率、修改率、重写次数）丰富且自然
- LLM-as-a-Judge 准确率高但成本也高，作为最后一步

三阶段：

**M1-M2 规则路由**
```yaml
# router-svc/configs/routes.yaml
routes:
  - task: rfp_parse
    primary: { provider: anthropic, model: claude-sonnet-4 }
    fallback: [{ provider: openai, model: gpt-4o }]
  # ...
```
- 路由决策 = 任务画像 + 路由表 + 成本预算
- 每次调用记录：provider/model/task/prompt_tokens/completion_tokens/latency/error

**M3 隐式反馈**
- 收集信号：用户采纳/修改/重生成的次数
- 后台跑批：每日聚合成 provider × model × task 的质量分
- 路由表动态调整：低质量分降权、高质量分升权（用 rollout 灰度）

**M4+ LLM-as-a-Judge**
- 随机抽样 1% 请求，用 Claude 评估输出质量
- 输出：score 0-100 + 问题分类
- 接入实时路由决策

实施细节：
- 路由表存 PG（带 version），可热更新
- 每次调用结果落 router_call_logs 表（含 prompt 摘要、输出摘要、反馈）
- 反馈聚合离线跑（每日 03:00）
- A/B 框架：路由表可按租户灰度

退出条件：
- 规则路由明显劣于人工选择（< 70% 满意度）
- 隐式反馈信号噪声过大（误判多）

## Q3：私有化是否提供模型

**推荐：默认不提供模型，客户自备；提供 Ollama 一键部署脚本 + 模型推荐清单**

理由：
- 模型包 50-200GB，分发成本高、版本管理复杂
- 客户硬件差异大（有的有 8xA100，有的只有 24GB 显存）
- 合规：客户数据出域是大忌
- 灵活性：客户可用任何 OpenAI 协议的服务（自托管 + 商用 API）

实施细节：
- 私有化部署包：helm chart + Ollama StatefulSet + 离线模型加载脚本
- 推荐配置（最低）：
  - GPU：1x RTX 4090 (24GB) 或 1x A10 (24GB)
  - 推荐模型：Qwen2.5-72B-Instruct-AWQ（4-bit 量化，~40GB 显存）
  - 备选：DeepSeek-V3-Distill-Qwen-32B（~20GB）
- 备选模型：vLLM / TGI 自部署
- 完全离线场景：先下载模型权重，离线后导入

提供模型（备选）的场景：
- 客户机器配置低于最低要求
- 客户要求"开箱即用"
- 提供方式：可选附加包，按订阅收费

退出条件：
- 客户机器普遍配置高（> 48GB 显存）
- 私有化市场规模大到值得自建模型分发

## Q4：Word 导出格式

**推荐：v1 仅支持 .docx（Office Open XML）；ODF/WPS 看 M3 反馈**

理由：
- .docx 是国内投标市场事实标准（90%+ 招标文件要求）
- .docx 生态成熟（docx 库 + Word/WPS 都能打开）
- ODF 主要欧洲政府用，国内几乎无
- WPS 是 .docx 超集（自家格式基本是 .docx 改名），无需单独支持

实施细节：
- 使用 github.com/nguyenthenguyen/docx 或类似 Go 库
- 借鉴 yibiao 的 exportService 设计：Markdown → AST → docx blocks
- Mermaid 渲染：优先 mermaid.ink（在线），降级本地 mmdc CLI
- 图片：嵌入 docx（base64），大图走外链

备选（看反馈）：
- 客户反馈大量 WPS 兼容问题 → 加 WPS 导出预设（页边距/字体调整）
- 政府/事业单位要求 ODF → 加 ODF 导出

退出条件：
- > 10% 客户主动要求 ODF
- WPS 兼容问题反复出现

## Q5：审计 agent 模式默认状态

**推荐：默认关闭，UI 显式按钮"深度审计"开启；M3 评估后考虑默认开启 normal、自动启用 agent**

理由：
- agent 模式（OpenCode 子进程）单次成本 5-10 倍 normal
- 用户对"为什么这次贵 10 倍"需要明确知情
- 普通项目 normal 已足够（90% issue 能抓到）
- 复杂项目 / 关键标段 / 客户主动要求 → 开 agent

实施细节：
- UI 默认勾选：normal 模式
- 高级选项：复选框"启用 agent 深度审计" + 预估成本提示
- 路由层：agent 模式强制走最强模型（Claude Opus 4 或等效）
- 配额保护：agent 模式单独计费（避免普通用户意外耗尽配额）
- 自动启用规则（M3 评估）：项目金额 > 100万 / 关键标段 / 失败率 > X

退出条件：
- 用户反复主动开启 agent（> 50%）
- normal 模式 issue 漏检率被证明过高

## Q6：行业模板市集冷启动

**推荐：v1 内部预置 6 个行业模板；M3 开放 UGC；M4 引入付费模板**

6 个预置行业（按国内投标市场份额选）：
1. **信息化系统集成**（最大类，IT 服务/系统集成/软件采购）
2. **政府采购**（政企客户，规则严格）
3. **工程 EPC**（基建/房建/市政）
4. **咨询服务**（可研/监理/审计）
5. **设备采购**（硬件为主，技术参数密集）
6. **教育培训**（课程/培训服务）

每个模板包含：
- 章节结构（3 级大纲 + 子节点）
- 必备模块清单（资质/案例/团队/技术方案/商务报价）
- 评分点对应表
- 必答条款清单
- 废标项高频黑名单
- AI 写作风格指南

实施细节：
- 模板存 PG：templates 表 + template_sections + template_rules
- 模板应用：用户创建项目时选模板，自动生成默认大纲
- 模板可被复制 + 改（私有模板）
- 模板可发布到市集（脱敏后）

M3 UGC：
- 模板作者：完成 KYC + 历史标数 ≥ 5
- 审核机制：内部审核 + 用户举报
- 收益：模板被使用时分成（10-30%）

M4 付费模板：
- 行业专家定制模板
- 价格：99-999 元/份

退出条件：
- 用户大量要求特定行业模板（说明市场真实需求）
- 模板数据积累到可统计阶段

## Q7：Web 与桌面数据同步冲突

**推荐：以云端为权威源；本地为缓存；冲突采用"云端优先 + 冲突列表"**

理由：
- 云端多设备协作场景更复杂，桌面用户少
- 桌面是"离线备份 + 本地加速"角色，不应成为权威
- 简单策略用户更易理解："我的标书在云端，桌面只是方便本地看"

冲突类型与处理：
| 冲突 | 处理 |
|---|---|
| Web 改、桌面未改 | 桌面下次同步时拉取 Web 版本 |
| 桌面改、Web 未改 | 桌面同步时推送到 Web |
| 双方都改 | 冲突！以 Web 为准，桌面改动进"冲突列表"待人工处理 |
| 桌面完全离线编辑 | 同步时按时间戳判断，旧的覆盖新的 |

实施细节：
- 桌面本地 SQLite 缓存 + 同步队列
- 同步协议：增量同步（按 version + last_modified）
- 冲突 UI：桌面端弹窗"Web 端有新版本，你的改动如何处理？"
  - 选项 A：放弃本地改动，加载 Web 版本
  - 选项 B：保留本地改动，作为新版本覆盖 Web（会覆盖 Web 协作内容）
  - 选项 C：手动合并（v2 实现）
- 同步频率：在线时每 30 秒增量；离线时累积
- 冲突日志：所有冲突写 audit_logs

退出条件：
- 桌面用户数 > 20%（说明桌面成为主战场）
- 冲突率 > 5%（说明规则需要调整）
