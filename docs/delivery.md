# 二、MVP 里程碑

## M1: 核心解析能力（Alpha）

**目标**：实现 RFP 文件解析与章节结构生成

| 功能 | 完成标准 |
|---|---|
| RFP 上传 | 支持 PDF/DOCX 格式上传，单文件 ≤ 50MB |
| 文本解析 | 提取标题、段落、表格、列表结构 |
| 需求识别 | 识别并标注技术需求、服务需求、资质要求 |
| 章节生成 | 基于 RFP 生成 3 级章节大纲，可人工调整 |
| 章节确认 | 用户确认章节结构后进入生成阶段 |

**交付物**：D-01, D-02, D-04, D-05, D-10, D-11

## M2: 内容生成能力（Beta）

**目标**：实现章节内容自动生成与图表渲染

| 功能 | 完成标准 |
|---|---|
| 章节生成 | 支持正式/简洁两种文风，字数满足最低要求 |
| 图表渲染 | 支持流程图、柱状图、饼图、表格四类图表 |
| 证据关联 | 自动关联证据文件到对应章节 |
| 进度展示 | 实时展示生成进度（章节 + 图表） |
| 内容导出 | 支持导出为标准 .docx 格式 |

**交付物**：D-03, D-20, D-21, D-13, D-15

## M3: 审计与优化（RC）

**目标**：实现 AI 自审与人工复核机制

| 功能 | 完成标准 |
|---|---|
| AI 审计 | 完整性、一致性、格式规范三维审计 |
| 问题定位 | 精确定位问题位置，提供修改建议 |
| 人工复核 | 支持人工确认/驳回审计结果 |
| 重新生成 | 支持单章节重新生成 |
| 评分预估 | 基于响应度给出评分预估 |

**交付物**：D-14, D-16, D-22

## M4: 知识增强（GA）

**目标**：实现知识库检索与向量匹配

| 功能 | 完成标准 |
|---|---|
| 知识库上传 | 支持企业资质、案例、证书等文档上传 |
| 向量索引 | 支持语义相似度搜索 |
| 证据推荐 | 自动推荐相关证据 |
| 图表双向匹配 | 图表可被语义检索，内容可被图表佐证 |

**交付物**：D-12, D-16（完整测试报告）

# 三、验收标准

## 3.1 功能验收

| 编号 | 验收项 | 验收条件 | 测试方法 |
|---|---|---|---|
| V-01 | RFP 上传 | 支持 PDF/DOCX，上传成功率 ≥ 99% | 上传 10 份不同格式 RFP |
| V-02 | 解析准确率 | 关键信息（招标单位、截止日期、预算）提取准确率 ≥ 95% | 与人工标注对比 |
| V-03 | 章节生成 | 生成章节结构与 RFP 章节匹配度 ≥ 90% | 5 份 RFP 测试 |
| V-04 | 内容字数 | 生成内容字数 ≥ 最低要求的 90% | 随机抽样 20 章节 |
| V-05 | 图表渲染 | 图表类型支持率 100%，渲染成功率 ≥ 95% | 10 类图表测试 |
| V-06 | 导出格式 | 导出 Word 格式兼容性：WPS/Office 2019+ 正常打开 | 多环境测试 |
| V-07 | 审计准确率 | 审计问题检出率 ≥ 85%，误报率 ≤ 15% | 100 个已知问题测试 |
| V-08 | 并发处理 | 10 个并发任务正常运行，无数据冲突 | 压力测试 |

## 3.2 性能验收

| 编号 | 验收项 | 指标 | 测试方法 |
|---|---|---|---|
| P-01 | API 响应时间 | P95 ≤ 500ms | 1000 次请求压测 |
| P-02 | RFP 解析速度 | ≤ 30s / 百页 | 100 页 RFP 测试 |
| P-03 | 章节生成速度 | ≤ 120s / 千字 | 计时测试 |
| P-04 | 图表渲染速度 | ≤ 10s / 图 | 10 类图表计时 |
| P-05 | 导出生成速度 | ≤ 60s / 百页 | 100 页文档导出 |
| P-06 | 向量检索延迟 | P95 ≤ 200ms | 1000 次检索压测 |

# 六、后续维护

## 6.1 维保期

- **标准维保期**：交付后 3 个月
- **维保范围**：缺陷修复、安全补丁、功能答疑
- **响应时间**：严重问题 4h，普通问题 2 工作日

- **大版本**（功能重构）：提前 1 个月通知，提供迁移文档

- **热修复**（安全/严重 Bug）：即时通知，24h 内升级
